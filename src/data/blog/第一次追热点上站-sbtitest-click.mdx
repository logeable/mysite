4 月 10 日早上七八点，我刚醒，随手打开朋友圈，结果连续看到两个人都在发 `SBTI` 人格测试结果。

我当时的第一反应不是“这是什么测试”，而是“这个词我之前完全没听过，它可能是刚爆出来的”。

于是我立刻打开电脑，先查了下 Google Trends。结果很直接，这个词在 24 小时内的搜索量明显暴涨，几乎就是一夜之间冒出来的。我又顺手去 Google 搜了一下，发现 `sbti` 的 SERP 首页几乎没有什么像样的独立站结果。再往下翻，我还看到有人把 SBTI 的镜像站代码放在了 GitHub 上。

作为技术人，我当时一下就意识到，这可能是一个可以立刻动手去追的热点。

## 从发现词到站点跑起来，不到半小时

确认这个词值得试之后，我做的事情非常简单粗暴：

1. 找可注册域名
2. 选一个能直接上线的部署方式
3. 先把站跑起来，再说后面的优化

我打开 NameSilo，纠结了大概 10 分钟，最后选了 `sbtitest.click`。原因也很现实，因为 `sbti` 相关的常见二级组合基本都已经被注册掉了，这个域名已经算是我当时能快速拿下、又还说得过去的选择。

买完域名之后，我直接把站部署到 Cloudflare Workers。因为当时已经有现成的镜像站参考，再加上部署链路足够轻，整个过程快得有点不真实，不到半小时，站就已经能访问了。

上线之后，我第一时间在自己的站上做了一次测试，截了图发朋友圈。我很“克制”地把网址贴了上去，但没有直接说这是我自己做的。随后我也把截图发到了小红书，图里同样带了网址。

这是我第一次做这种“追热点就地上站”的事情，所以当时的情绪其实很复杂。

一方面很兴奋。因为前几天我还在学怎么用 Semrush、Google Trends 挖词，结果今天就碰到了一个真的能实践的词，而且不是纸上谈兵，是可以立刻上手干的那种。

另一方面又有点忐忑。我会想：

- 这种做法会不会不太道德？
- 会不会涉及侵权？
- 公开发出来之后会不会招来非议？

最终，还是“先做出来试试”的技术冲动压过了我心里的那层顾虑。
这里也顺带说明一下，我理解的 `SBTI` 更偏抽象和娱乐化，它和 MBTI 还是有明显区别的。

## 既然决定做，就尽量做得像回事一点

决定试试之后，我就没打算只停留在“镜像一份页面然后等流量”这个层面。

我当时的想法很简单：既然已经动手了，那就尽量把这个机会利用到最大，看看自己到底能拿到多少流量。

我先借助 Codex 对现有页面做差异化，补全一些 GitHub 镜像站里没有的东西。原始镜像基本就是纯前端静态页面，功能和结构都很薄，所以比较容易做补强。

大概花了一两个小时，我主要做了几件事：

- 优化首页的 `title`、`description`、`keywords`
- 补充 `og` 信息
- 完善 `h1`、`h2` 等基础结构
- 增加大量内链
- 在测试结果页加上更方便的分享能力，提升裂变概率

做到这里的时候，我觉得“第一阶段”其实已经完成了，也就是：

`先上线 -> 先可用 -> 再把最基础的 SEO 和分享链路补起来`

## 搜索收录完全不熟，但我还是边做边补

接下来我遇到的就是一个很现实的问题：

站点上线了，但新站到底怎么提高曝光？

那时候我其实还不知道新站应该怎么被 Google 收录，于是开始一边搜、一边学、一边补动作。

我先打开 Google Search Console，把站提交了进去，也提交了 sitemap，但它的状态一直显示“无法抓取”。直到现在还是这样，如果有朋友正好懂这一块，也欢迎指点我一下这个问题到底该怎么排查。

同时，Search Console 还提示相关数据需要一天后才能看到，所以它在“即时反馈”这件事上对我帮助并不大。

而 Cloudflare Analytics 虽然能看到一些流量数据，但颗粒度不够细。这个时候我突然想起之前研究过的 PostHog，于是马上动手把 `sbtitest.click` 接了进去。

结果接进来没多久，我就发现站其实已经有流量了。

几乎每分钟都有事件数据进来。

我当时猜测，第一波流量大概率来自朋友圈，因为那个时间点小红书还只有 6 个曝光。后来我去看 PostHog 的 referring data，也基本验证了这个判断：来自微信的用户占了几乎一半。

这一刻对我的刺激其实挺大的。

因为它让我第一次非常直观地感受到：一个你早上才发现的词，一个你半小时搭起来的小站，真的有可能在当天就开始收到自然访问。

## 下午开始边验证流量，边拿这个项目练 Next.js

下午吃完饭之后，我突然又冒出另一个想法：干脆趁这个机会，把这个站顺手拿来练一下 Next.js。

于是我就在 Codex 的帮助下，把原来纯静态的 `html + js + css` 项目改造成了 Next.js 项目。因为当时并不需要真正的后端能力，所以部署方案依然走静态导出，也就是 `output: "export"` 这套模式。同时，我还顺手补了英文模式。

现在回头看，这个决定其实有点“边打仗边换枪”，不算特别稳妥，但它确实让我在一个很真实的场景里把 Next.js 过了一遍。

而且到了下午，流量确实也开始明显往上走了。

- 上午的 UV 还只是几十
- 到下午三点多，大概到了 200
- 快到下班的时候，已经接近 500

我觉得这里面有几个动作是起作用的：

1. 我下午会不时去小红书回复别人“`sbti` 在哪里测”的帖子
2. 我在 Wikipedia 补了一个外链，这里大概贡献了 50 个 UV
3. 我也在 X 上发了相关内容

不过 X 的结果相当现实，几乎没给我什么曝光。直到我写这篇文章的时候，X 带来的 UV 也只有 2。

更扎心的是，我后来在下班前发现，有些大 V 几乎和我上站是同一时间开始发和 SBTI 相关的内容，但他们的帖子浏览量动不动就是几万。

这也让我第一次很直接地意识到，热点词本身只是一个入口，分发能力、外链能力、账号能力和执行速度其实都非常关键。

## 晚上是流量高峰，我把排行榜也补上了

到了晚上七八点，流量进入高峰期。

PostHog 的 live events 一直在刷，那个页面我盯着看了很久。对第一次自己做热点站的人来说，这种“实时有人在点、有人在测、有人在跳转”的感觉真的非常上头。

看着看着，我又想到一个问题：用户做完测试之后，很可能不只想看自己的结果，还会想知道“大家都测成了什么”。

于是我又继续补功能，把排行榜做了出来，数据存到 Cloudflare D1 里。

等到晚上 12 点左右，当天 UV 大概到了 800 多。

说实话，作为第一次追热点上站，这个结果已经让我挺开心了。

## 但看到别人拿到百万流量时，还是被狠狠教育了一次

本来我对这一天的结果已经比较满意了，直到后来我在 X 上看到哥飞发了一张截图，说他们群友拿到了百万流量。

那一瞬间我的情绪非常复杂。

有震惊，也有失落，还有一点难过。

因为你会突然意识到，自己努力了一整天，虽然确实做成了一些事情，但从更大的结果看，可能在方向、方法、速度或者分发策略上，早就已经落后了很多。

更重要的是，我在这之前甚至没有充分意识到自己的问题到底在哪里。

也正因为这个冲击，我才决定把整个过程写下来。与其只是感叹“别人为什么做得更好”，不如把自己这一天的执行路径、卡点和教训完整复盘出来，至少给未来的自己留下一份 SOP。

## 这次暴露出来的几个问题

### 1. 第一次追热词上站，流程完全不熟

虽然最后确实把站做出来了，但整个过程更多是“想到什么补什么”，而不是沿着一个成熟流程在跑。

这会导致很多动作做得比较晚，比如：

- 收录提交
- 统计接入
- 页面矩阵扩展
- 外链分发

如果这些动作能在更靠前的阶段系统化执行，理论上结果应该会更好一些。

### 2. 最基础的 SEO 其实并没有真正实践过

我之前知道一些 SEO 的概念，也看过一些内容，但这次是第一次在真实站点上快速实践。

所以你会发现，我虽然知道要补 `title`、`description`、`h1`、内链，但这些动作到底应该怎么排序、做到什么程度、哪些是最优先的，其实都还是模糊的。

### 3. 对 Next.js 还不够熟

这次改造项目，基本上是靠 Codex 帮我一路往前推的。能跑起来是一回事，但代码里肯定还有不少重复内容和不够干净的地方。

这也提醒我，工具能帮你提速，但基础熟悉度不够的时候，后面维护和扩展的成本还是会落回自己身上。

### 4. 还不知道怎么稳定地做更多外链

这次我能想到的外链渠道并不多，主要还是：

- 微信朋友圈
- 小红书
- Wikipedia
- X

这些动作确实带来了一些流量，但整体上还是很随机，也缺少体系感。尤其当你想进一步放大流量时，外链和分发的能力很可能会直接决定上限。

## 这次得到的几个教训

第一，追热点最重要的不是“想得明白”，而是“发现后立刻做出来”。

这次我能拿到一点结果，一个很关键的原因就是动作足够快。看到词，验证趋势，确认 SERP，注册域名，部署上线，整个链路没有拖太久。热点这件事本身就不等人，慢半天，可能就是另一个世界了。

第二，先上线比先完美更重要。

如果我一开始就想把站做得很完整、很规范、很漂亮，那它很可能根本赶不上第一波流量。真正更合理的方式是先拿到可访问版本，再快速补 SEO、分享、统计和内容结构。

第三，热点站不只是技术问题，还是分发问题。

技术让我能快速上线，但真正决定流量上限的，往往是：

- 你能不能快速获得第一波种子用户
- 你能不能做出更适合分享和传播的结构
- 你能不能把站点尽快暴露到更多平台和外链里

第四，复盘比短暂兴奋更重要。

800 多 UV 对第一次来说当然值得开心，但如果不能把为什么拿到、为什么没拿到更多、下一次怎么做得更好这些问题讲清楚，那这次经验的价值就会很有限。

## 我给自己整理的新站 SEO 启动 SOP

这部分不一定完全正确，但至少是我这次做完之后，觉得以后每次追热点都应该优先检查的东西。

### 一、技术基础先补齐

这是保证 Google 能抓你的前提。

1. 添加 `sitemap.xml`，先把重点页面放进去
2. 确保 `robots.txt` 允许抓取
3. 页面必须有 `title` 和 `description`
4. 每个页面都要有明确的 `H1`
5. 基础的结构化内容和可索引链接要尽量完整

### 二、上线后尽快提交收录

1. 上线 10 分钟内提交 Google Search Console，有条件的话百度也提一下
2. 提交站点地图
3. 对重点页面手动请求索引
4. 尽快做第一批外链，让搜索引擎发现你

### 三、新站不要只做一个页面

这次我最大的感受之一就是，新站如果只有一个单页，其实很难建立主题相关性。

更合理的做法是围绕主题去做一个简单的内容 cluster。比如 SBTI 这种词，至少可以先铺出这样几类页面：

```txt
/sbti-test
/sbti-types
/sbti-vs-mbti
/what-is-sbti
/sbti-accuracy
/sbti-result
/sbti-leaderboard
```

这种结构更像一个完整站点，而不是一个只想蹭词的孤立页面。至少从直觉上讲，它更容易让搜索引擎理解你的主题覆盖范围。

### 四、把内链当成基础设施来做

每个页面之间要能互相连起来。

例如：

- `sbti-test` 链到首页、类型页、结果页、排行榜页
- 类型页再链到具体说明页
- 详情页之间彼此推荐

这样做的意义不只是用户跳转更顺，还在于它会让整个站更像一个有组织的内容网络，而不是零散页面拼起来的东西。

### 五、第一批外链一定要尽快做

我目前能想到、也实际做过的一些渠道包括：

- X
- Wikipedia
- Reddit
- 知乎
- 小红书
- 各类论坛和工具站

这里我还远远没有形成成熟方法，但至少我确认了一点：外链不是“站做完之后再考虑”的事情，而是上线第一天就该同步推进的动作。

### 六、持续优化点击率和扩展关键词

如果一个词已经起量，那后面要做的就不只是守着这个词本身，还要继续去扩展：

- 长尾词
- 对比词
- 解释型词
- 结果型词
- 用户会追问的问题词

同时也要继续优化标题、描述和分享文案，尽量提高点击率和传播率。

## 写在最后

以上这些，其实都只是我第一次追热点上站之后的一些粗糙反思。

它们不一定能构成对别人有普适意义的方法论，更谈不上建议。但对我自己来说，这一天很重要。

因为这是我第一次真正把“发现词 -> 迅速上线 -> 补 SEO -> 做分发 -> 看数据 -> 做复盘”完整走了一遍。

虽然最后的结果只是“一点点成果”，和那些真正吃到大流量的人比起来不算什么，但至少它让我不再只是旁观者，而是第一次真正下场做了一次。

我希望未来的自己还能记得这一天：

记得那种早上突然发现机会的兴奋，记得第一次看到实时流量刷新的上头，记得看到别人百万流量时的失落，也记得这些情绪最后都应该落到复盘、沉淀和下一次更快更稳的执行上。

如果以后我还能继续做热点站，我希望自己能比这次更熟练一点，也更有章法一点。

以上仅仅是我个人的一些记录和总结，仅供参考，不构成建议。
